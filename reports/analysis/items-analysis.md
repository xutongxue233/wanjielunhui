**分析时间**: 2026-02-09
**分析范围**: src/data/, src/stores/, src/types/, server/prisma/seed/

## 一、系统概述

万界轮回的装备道具系统是游戏的核心经济和成长系统,包含:
- **装备系统**: 5个部位的装备,支持品质分级和强化
- **道具系统**: 消耗品、材料、任务物品、货币
- **炼丹系统**: 丹药配方、炼丹材料、丹炉系统
- **背包系统**: 物品存储和管理
- **掉落系统**: 副本奖励和怪物掉落

### 2.3 装备属性系统

**基础属性**:
- `attack`: 攻击力
- `defense`: 防御力
- `hp`: 生命值
- `mp`: 法力值
- `speed`: 速度
- `critRate`: 暴击率 (百分比)
- `critDamage`: 暴击伤害 (百分比)
- `spiritualPower`: 灵力

**装备强化**:
- 每件装备有 `maxEnhanceLevel` (5-100)
- 每级提供 `enhanceStatsPerLevel` 的属性加成
- 强化上限随品质递增: common(5) -> legendary(30+)

**境界要求**:
| 境界 | 标识 |
|------|------|
| 无要求 | none |
| 练气期 | lianqi |
| 筑基期 | zhuji |
| 金丹期 | jindan |
| 元婴期 | yuanying |
| 化神期 | huashen |

### 2.4 装备特效系统 (仅后端)

后端装备支持 `specialEffect` 字段,定义装备的特殊效果:
- "攻击有15%概率冰冻敌人1回合"
- "暴击时触发雷霆,造成额外50%伤害"
- "受到致命伤害时,有20%概率保留1点生命"
- "每回合恢复5%最大生命值"

> **问题**: 前端装备定义没有 specialEffect 字段,特效系统未实现

## 五、背包与存储系统

### 5.1 背包结构

```typescript
interface Player {
  equipment: Record<EquipmentSlot, Equipment | null>;  // 6个装备槽
  inventory: InventoryItem[];  // 背包物品列表
}

interface InventoryItem {
  item: Item;
  quantity: number;
}
```

### 5.2 物品堆叠

- 消耗品: 最大堆叠 20-99
- 材料: 最大堆叠 50-999
- 货币: 最大堆叠 99999-999999
- 任务物品: 不可堆叠 (maxStack: 1)

### 5.3 背包操作

**现有功能**:
- `addItem`: 添加物品 (自动堆叠)
- `removeItem`: 移除物品
- `equipItem`: 穿戴装备 (自动计算属性)
- `unequipItem`: 卸下装备 (自动扣除属性)

**缺失功能**:
- 背包容量上限
- 物品排序
- 物品筛选
- 批量操作
- 快速使用
- 物品分解

## 七、系统问题与缺陷

### 7.1 严重问题 (P0)

1. **前后端数据不一致**
   - 前端35件装备,后端32件装备,ID不完全匹配
   - 前端44个道具,后端42个道具
   - 品质枚举不一致: 前端缺少 mythic

2. **装备特效未实现**
   - 后端定义了 specialEffect 字段
   - 前端装备接口和战斗系统未支持特效

### 7.2 功能缺失 (P1)

1. **背包系统不完整**
   - 无背包容量上限
   - 无物品排序/筛选
   - 无批量操作
   - 无物品分解系统

2. **炼器系统缺失**
   - 有炼器材料定义 (玄铁、寒冰精铁等)
   - 无炼器功能实现

3. **装备套装系统**
   - 装备定义有 setId 字段
   - 套装效果未实现

4. **装备镶嵌系统缺失**
   - 无宝石/符文系统
   - 无装备打孔

5. **物品交易系统**
   - 无玩家间交易
   - 无拍卖行

### 7.3 数值问题 (P2)

1. **掉落率偏低**
   - 高级物品掉落率普遍 5-10%
   - 可能导致玩家体验不佳

2. **部分副本无掉落**
   - 1-3章副本 drops 为空数组
   - 早期玩家缺乏装备来源

3. **炼丹材料获取困难**
   - 部分材料无掉落来源 (如 spirit_grass, morning_dew)
   - 配方材料与实际可获取材料不匹配

### 7.4 设计问题 (P2)

1. **头盔/鞋子装备数量不足**
   - 前端仅6件头盔、6件鞋子
   - 境界覆盖不完整

2. **uncommon品质鞋子缺失**
   - 鞋子从common直接跳到rare

3. **炼丹材料ID不一致**
   - 配方引用的材料ID与实际材料ID不匹配
   - 如: spirit_grass 在配方中引用但 items.ts 中是 lingzhi

## 九、优化建议

### 9.1 高优先级

1. **统一前后端数据**
   - 同步装备和道具定义
   - 统一品质枚举 (添加 mythic)
   - 统一物品ID命名规范

2. **补全早期副本掉落**
   - 为1-3章添加材料掉落
   - 添加低级装备掉落

3. **实现装备特效系统**
   - 在战斗系统中解析 specialEffect
   - 添加特效触发逻辑

4. **统一炼丹材料ID**
   - 修复配方引用的材料ID
   - 添加材料获取途径

### 9.2 中优先级

1. **完善背包系统**
   - 添加背包容量 (可升级)
   - 实现物品排序和筛选
   - 添加物品分解功能

2. **补充装备种类**
   - 增加uncommon品质鞋子
   - 补充中间境界装备
   - 完善符箓系统 (前端)

3. **实现套装系统**
   - 设计套装效果
   - 实现套装属性计算

4. **添加炼器系统**
   - 实现装备锻造
   - 实现装备升阶

### 9.3 低优先级

1. **添加装备镶嵌系统**
   - 设计宝石/符文系统
   - 实现装备打孔

2. **添加物品交易系统**
   - 玩家间交易
   - 拍卖行系统

3. **添加物品图鉴系统**
   - 收集进度追踪
