# 项目状态命令

你是项目管理助手，负责汇报当前项目状态。

## 执行内容

### 1. 代码状态

```bash
git status
git log --oneline -5
```

### 2. 构建状态

```bash
pnpm build 2>&1 | tail -20
```

### 3. 任务状态

```
TaskList()
```

### 4. 迭代状态

读取 `reports/iteration/latest.json`（如存在）

### 5. 问题状态

读取 `reports/iteration/latest/issues.json`（如存在）

## 输出格式

```markdown
# 项目状态报告

时间: YYYY-MM-DD HH:mm

## Git 状态

当前分支: main
未提交修改: 5 个文件
最近提交:
- abc1234 feat: 添加好友系统
- def5678 fix: 修复登录问题

## 构建状态

前端: 通过 / 失败
后端: 通过 / 失败
错误数: 0

## 任务状态

| 状态 | 数量 |
|------|------|
| 待处理 | 3 |
| 进行中 | 1 |
| 已完成 | 5 |

当前任务: [任务名称]

## 迭代状态

状态: 进行中 / 已完成
轮次: 2/10

问题统计:
- P0: 0 (已清零)
- P1: 2 (剩余)
- P2: 5

## 待处理问题

| ID | 严重度 | 描述 |
|----|--------|------|
| ISSUE-001 | P1 | API 返回 500 |
| ISSUE-002 | P1 | 登录状态丢失 |

## 建议下一步

1. 处理 P1 问题: ISSUE-001
2. 继续迭代循环
3. ...
```

## 快速诊断

如果发现问题，给出快速建议：

| 状态 | 建议 |
|------|------|
| 构建失败 | 运行 `/quick-fix` |
| P0 问题存在 | 立即运行 `/iter-fix` |
| 任务堆积 | 检查阻塞原因 |
| 长时间无提交 | 检查工作进度 |

## 退出条件

- 状态报告输出完成
