```markdown
## 快速开始

### 开发新功能
```
/build 添加好友系统
```
或分步执行：
```
/plan-feature 好友系统
/dev-feature  （基于设计文档实现）
```

### 修复问题
```
/quick-fix 登录页面报错 Cannot read property 'username'
```

### 自动化测试
```
/iter-loop
```

### 查看状态
```
/status
```

## 工作流推荐

### 新功能开发流程
1. `/plan-feature` - 策划设计
2. `/dev-feature` - 代码实现
3. `/iter-devtools-test` - 功能验证
4. `/review` - 代码审查

### 问题修复流程
1. `/quick-fix` - 快速修复
2. 或 `/iter-loop` - 完整迭代

### 游戏内容开发
1. `/content-data` - 配置数据
2. `/content-story` - 编写剧情
3. `/balance` - 数值平衡

## 注意事项

1. 大部分命令支持无人值守执行
2. 发现问题会自动尝试修复
3. 报告输出到 `reports/` 目录
4. 使用 DevTools MCP 进行浏览器自动化

## 获取帮助

如需详细了解某个命令，直接询问：
> "/build 命令怎么用？"
> "如何添加新的游戏物品？"
```
